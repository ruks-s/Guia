O que é CSS?

É uma linguagem utilizada para a estilização de uma linguagem de marcação (HTML por exemplo), que tem como tradução Cascading Style Sheets (Folhas de Estilo em Cascata) - e bom, apenas o nome da linguagem já explicita partes de como ela é - "style sheets" visto que é a responsável por estilizar a página, e "cascading" por conta do seu comportamento de aplicação dos estilos (de seu funcionamento basicamente, que funciona "em cascata") que será abordada mais a frente.

Por agora, é bom sabermos a sintaxe básica que todo código em CSS vai se basear (em maioria), que é o seguinte: 

                    
                        selector { 
                            propriety: value;
                        }

Onde, basicamente, o selector corresponde ao elemento selecionado (a forma de escolha varia bastante, será abordado posteriormente), propriety é correspondente ao que será personalizado, e value o valor da propriedade mudada, vale destacar o ";" sempre no final de cada valor para cada propriedade - e lembrando que essa é a forma base, tem-se outras formas de utilizar o CSS que podem fugir levemente aparentemente dessa base, mas sempre vão estar relacionadas a essa sintaxe.

Formas de Utilizar CSS

Bom, existem 3 formas principais de estilizar os elementos - quando digo "formas", são jeitos diferentes de utilizar códigos no estilo CSS para personalizar algum elemento HTML, podendo esses códigos, estarem dentro ou fora do documento HTML. As 3 principais são:

  • Inline
  • Internal
  • External

Inline

Consiste em colocar o código CSS, juntamente do atributo "style" dentro do próprio elemento, como descrito abaixo: 

                        
                            <element style="color: green;">conteúdo</element>

Como mostrado acima, o estilo é aplicado da forma mais direta possível no elemento - literalmente dentro do mesmo - esse tipo de estilização é bom em casos que você precisa mudar coisas pontuais, visto que eles tem uma prioridade superior às outras formas de estilização (isso será visto posteriormente em Ordem de Prioridade).

Internal

Consiste em colocar a estilização (dessa vez, seguindo a sintaxe colocada anteriormente), logo na cabeça do documento HTML (<head>), com a utilização do elemento <style> - apesar de possibilitar uma customização mais ampla do o que o anterior, os problemas relacionados a esse modo de estilização é a parte de reaproveitamento das estilizações em projetos grandes, além de aumentar bastante o tamanho do documento HTML.

                        
                            <head>
                                <style>
                                    element {
                                        propriety: value;
                                    }
                                </style>
                            </head>
                            <body>
                                <element>
                            </body>

External

Tipo de estilização mais utilizado, que consiste em fazer um documento .css focado apenas na estilização do site - é o mais utilizado por conta da facilidade de manutenção que ele fornece, além do reaproveitamento facilitado e melhor organização em relação a divisão de funções de cada documento.

                        
                            documento.css 
                                selector {
                                    propriety: value;
                                }

Seletores

Seletores são os responsáveis por especificar qual ou quais elementos serão alvos da estilização descrita no código CSS em questão, isso pode ser feito de diversas maneiras (que serão abordadas posteriormente), seja diretamente escrevendo o próprio elemento como sendo o seletor, ou até utilizando graus de parentesco dos elementos HTML para tal (para fins de facilidade, consideraremos todos os códigos basicamente escritos como se fossem para um documento CSS a parte do HTML). Tipos de Seletores:

  • Seleção Direta
  • Seleção por ID
  • Seleção por Classe
  • Seleção por Atributo
  • Seleção por Pseudos
  • Seletor Universal
  • Descendant Combinator
  • Child Combinator
  • Next-sibling Combinator
  • Subsequent-sibling Combinator
  • Ordem de Prioridade

Seleção Direta

Consiste em utilizar o nome próprio de um elemento como seletor, possibilitando, portanto, uma generalização na personalização de diversos elementos iguais. Tem sua sintaxe como: 

                        
                            nome do elemento {
                                propriedade: valor;
                            }

Seleção por ID

Como o nome sugere, é uma seleção baseada no ID de um elemento específico (lembremos que ID é algo único, apenas 1 elemento tem ele por vez) esse tipo de seleção é especialmente bom para casos específicos, onde talvez o elemento já possa ter um classe geral, e tals. Se caracteriza por um "#" precedido do valor do ID: 

                        
                            #valor do id {
                                propriedade: valor;
                            }

Seleção por Classe

É provavelmente o sistema mais comum de vermos, que se baseia, logicamente, nas classes dos elementos, esse método é bom principalmente por conta da diversidade e especificação conjunta que ele dá para a estilização dos elementos, sendo caracterizado por um "." antes do valor da classe: 

                        
                            .valor da classe {
                                propriedade: valor;
                            }

Seleção por Atributo

Logicamente, como o nome sugere, se baseia em alguma característica do atributo para a seleção dos elementos que serão customizados, tendo sua sintaxe base sendo: 

                        
                            nomedoelemento[atributo] {
                                propriedade: valor;
                            }

Além dessa forma mais básica de seleção, tem-se outras formas que se baseiam no mesmo princípio, mas adicionando alguma nuance, segue abaixo alguns exemplos: 

                        
                            elemento[atributo~=valor] {} -> atinge todos os elementos que tem o atributo com o valor especificado
                            elemento[atributo^=valor] {} -> atinge todos os elementos que tem o atributo com o inicio que é o valor especificado
                            elemento[atributo$=valor] {} -> atinge todos os elementos que tem o atributo com o final que é o valor especificado

Seleção por Pseudos

São seletores condicionados a características específicas do elemento (estado, condições especiais, etc), caracterizadas por um ":" ou "::" entre o elemento e a pseudo-classe/pseudo-elemento. Sintaxe Base: 

                        
                            elemento:pseudoclasse {
                                propriedade: valor;
                            }

Existem diversos, quase infinitos pseudo-seletores, mas, tem alguns que são bem mais recorrentes que valem a pena ter eles na cabeça: 

                        
                            elemento:active {} -> personaliza o elemento quando ele ta ativado
                            elemento:hover {} -> personaliza o elemento quando ele tem um mouse por cima
                            elemento:focus {} -> personaliza o elemento quando ele é focado por teclado
                            elemento:visited {} -> personaliza o elemento quando ele já foi visitado 
                            elemento:checked {} -> personaliza o elemento quando ele foi checkado

Seletor Universal

Como o próprio nome sugere, aplica uma estilização a todos os elementos do documento HTML, apesar de ter sua utilidade, por ter o mais baixo nível de prioridade dentre os seletores, ele acaba ficando meio de escanteio quando fazemos um site, mas bom saber que existe - caracterizado por um "*" antes do código CSS: 

                        
                            * {
                                propriedade: valor;
                            }

Descendant Combinator

A partir de agora, entraremos nos seletores baseados em relações de parentesco presente entre os elementos HTML, começando pelo Descendant, que se baseia em escolher apenas os elementos selecionados em segunda instância que sejam filhos, netos, etc, do primeiro elemento colocado, não precisa-se de nada entre os elementos, exemplo: 

                        
                            elemento1 elemento2 {
                                propriedade: valor;
                            }

Nesse caso em questão, só serão estilizados, os elemento2 que são, independentemente da profundidade (filhos, netos, etc), descendentes do elemento1.

Child Combinator

Baseia-se em selecionar elementos que são filhos diretos de um outro elemento, sendo caracterizado pela presença de um ">" entre os elementos: 

                        
                            elemento1 > elemento2 {
                                propriedade: valor;
                            }

Ou seja, só serão estilizados os elemento2 que são filhos direto do elemento1.

Next-sibling Combinator

Seleciona o elemento-irmão imediatamente seguinte (ou seja, atinge, a priori, apenas 1 elemento), caracterizado pela presença de um + entre os elementos, exemplo: 

                        
                            elemento1 + elemento2 {
                                propriedade: valor;
                            }

Será estilizado apenas o elemento2 que é irmão imediatamente seguinte ao elemento1.

Subsequent-sibling Combinator

Seleciona todos os irmãos posteriores ao elemento especificado, caracterizado pela presença de um ~, exemplo: 

                        
                            elemento1 ~ elemento2 {
                                propriedade: valor;
                            }

Apenas serão selecionados todos os elementos2, que sejam irmãos do elemento1, e que estejam abaixo dele na organização da página.

Ordem de Prioridade

Agora que temos conhecimento sobre os diversos seletores que existem, precisamos saber como a estilização CSS se comporta quando temos por exemplo, um seletor de id e um universal aplicado a um mesmo elemento - isso vai ser definido pela prioridade de cada tipo de seletor. Tem sim um sistema bem complexo por trás de como é definido essa escolha, mas, a termos de simplificação, podemos utilizar essa regra: Inline Style > ID > Class > Element > Universal.

Porém, como nem tudo são flores, tem algumas coisas que fazem com que essa ordem de prioridade seja contornada, como por exemplo, a existência de uma chave de importância (!important) em alguma estilização, que faz com que ela seja prioridade quase que em todo caso. Além disso, tem-se outros fatores analisados pelo navegador que faz ele selecionar ou não outro tipo de estilização, mas a regra geral é:

1. Relevância: olhando todas as estilizações aplicadas em cima de um elemento, e analisando certas coisas que podem estar em vigor - como regras baseadas nas características do dispositivos ou da tela.

2. Após isso, o algoritmo considera a origem e a importância: isto é, considera se o estilo é o padrão do navegador ou do usuário - e a origem, considerando regras marcadas por !important, acima das outras regras independente da origem.

3. Após isso, é analisado a especificidade/prioridade falada anteriormente.

4. E se no fim, tiver 2 estilizações com a mesma prioridade aplicadas em cima de um elemento, a que aparece por último no código CSS, será aplicada (exemplo clássico, é entre o estilo de CSS internal e external - ambas tem o mesmo peso em questão de prioridade, o que define qual será aplicada, é qual aparece por último dentro do <head> *no caso do internal, é quando aparece o <link>
