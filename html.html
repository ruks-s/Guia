Principais Elementos Presentes nos Metadados

- html

Apesar de soar meio estranho, sim, é um elemento presente na estrutura de uma página, onde, literalmente, TODO o código está presente dentro dele, geralmente é atribuído um único atributo a ele, que é o da linguagem da página, porém, nunca esquecer, que ele inicia o código (logo depois da declaração - será vista mais pra frente em Explicações/Conceitos Importantes) e dita o término do código.

Exemplo:
            
              <html lang="en"> (onde nesse caso, a linguagem utilizada seria inglês)
                código da página inteira
              </html>

- head

É basicamente o elementos no qual abrigará todo seu código feito dos metadados do documento em questão.

Exemplo:
            
              <head> 
                conteúdo/códigos relacionados aos metadados (será aprofundado posteriormente)
              </head>

- meta

Um elemento vazio, e provavelmente o mais importante elemento presente dentro dos metadados de um documento, por basicamente definir tudo que se espera encontrar dentro do mesmo, podendo ter diversos atributos específicos para ele, lembrando que, cada atributo, por assumir geralmente 1 valor por vez, cria a necessidade de em apenas 1 documento, se ter diversos elementos de <meta/>. Veja mais em: Atributos para Metadados

Exemplo:
              
                <meta atributo="valor"/>

- title

Como o próprio nome já diz, determina o título do seu site, não visualmente, mas por exemplo, para o navegador, é uma das formas na qual a busca pode ir parar no seu site.

Exemplo:
              
                <title>seu título</title>

- link

Apesar de não ser um elemento vazio exclusivo apenas para a parte de metadados, é muito presente na mesma, tendo como principal função nesse caso, definir o estilo (principalmente com css), do site em questão, através de um link (como o próprio nome sugere) - sua função mais geral, será aprofundada posteriormente durante Principais para a Formação Além da Escrita.

Exemplo:
              
                <link rel="" href=""/> -> tanto o atributo "rel" quanto o atributo "href" será visto posteriormente em Atributos.

Principais Elementos Presentes na Escrita do Site

- h 1-6

Inicialmente, pode parecer estranho a escrita do código, porém, é simples de entender, é basicamente o que dita os títulos e subtítulos de um site, em ordem de importância (e de tamanho geralmente), sendo o <h1> o com mais relevância, e o <h6> com menos relevância no site.

Exemplo:
              
                <hX>título/subtítulo</hX> -> onde X representa um número de 1-6, a depender da importância.

- p

Provavelmente o elemento mais utilizado na formação de sites (front-end), a grosso modo, é basicamente toda e qualquer escrita com exceção de situações especifícas como título, etc - mas é disparado o mais utilizado, tendo como um significado válido, tal qual sua letra representante: paragráfo.

Exemplo:
              
                <p>paragráfo (podendo ser texto, frase, etc)</p>

- a

O famoso "âncora", apesar de não ser algo exclusivo na formação de frases, uma das suas principais funções é justamente, a "indexização" de um link ou redirecionamento geral, em uma frase, palavra, etc, com o auxílio de um atributo, o href="", que será visto posteriormente em Atributos.

Exemplo:
              
                <a href="local de redirecionamento"> frase, palavra, etc...</a>

- ul

Utilizado para a criação de uma lista não ordenada, sendo os seus tópicos, definidos pelo elemento <li>, que será visto nos próximos tópicos ainda nessa seção.

Exemplo:
              
                <ul>
                  <li>conteúdo</li>
                  <li>conteúdo</li>
                  <li>conteúdo</li>
                </ul>

- ol

Utilizado para a criação de uma lista ordenada (1. 2. 3. ...)(em ordem de importância, de sequência lógica, etc), sendo os seus tópicos, definidos pelo elemento <li>, que será visto no próximo tópico.

Exemplo:
              
                <ol>
                  <li>conteúdo</li>
                  <li>conteúdo</li>
                  <li>conteúdo</li>
                </ol>

- li

Como citado anteriormente, é utilizado para fazer listas, sendo geralmente, filho do <ul> ou do <ol>. Veja mais sobre relação de "pai e filho" em: Explicações/Conceitos Importantes.

Exemplo:
              
                <ol> ou <ul>
                  <li>conteúdo</li>
                  <li>conteúdo</li>
                  <li>conteúdo</li>
                </ol> ou </ul>

- dt

Utilizado envolta de termos que serão descritos, explicados por um outro elemento: o <dd>, que será visto logo a diante.

Exemplo:
              
                <dt>elemento descrito</dt>

- dd

Usado para descrever algum elemento anterior a ele, englobado pelo <dt>, o conteúdo presente dentro desse elemento, por padrão, já fica afastado da margem.

Exemplo:
              
                <dt>elemento descrito</dt>
                  <dd>descrição</dd>

Principais Elementos Presentes no Site Além da Escrita

- img

Como o próprio nome sugere, é um elemento vazio utilizado para anexar uma imagem na site, utilizando-se diversos atributos para uma melhor experiência e aproveitamento de tal elemento, note que, obrigatoriamente, sempre se terá um atributo ao menos, visto que, o elemento sozinho, não é capaz de ter uma imagem por si só.

Exemplo>
              
                <img src="link da imagem"/> -> o atributo "src" será visto posteriormente em Atributos.

- input

Elemento vazio, utilizado para coletar dados dos usuários, podendo assumir diversos "tipos" (checkbox, escrita, etc), a depender dos valores de certos atributos acompanhados a ele.

Exemplo:
              
                <input type="button"/> -> o atributo "type" será visto posteriormente em Atributos, assim comos seus possíveis valores.

- link

Como citada anteriormente na seção de Metadados, tem como principal objetivo, a colocação de um link para alguma função específica, sendo majoritariamente, para a utilização de estilos, etc. Curiosidade: Google Fonts é um bom lugar para achar links para esse elemento.

Exemplo:
              
                <link rel="" href=""/> -> tanto o atributo "rel" quanto o atributo "href" será visto posteriormente em Atributos.

- script

Utilizado para executar ou carregar um código JavaScript na página, apesar de não ser a forma mais comum de se aplicar um código desses - geralmente é posto em outro documento e esse elemento serve apenas como "ativação" - é bom ter noção que se pode se escrever o código JavaScript dentro dele.

- button

Obviamente, é usado para formar um botão, onde, diferentemente do <input/>, por esse elemento não ser vazio, o torna mais "flexível" por assim dizer, no quesito de adicionar conteúdos nele. Geralmente utilizado junto do elemento <form>.

- audio

Autoexplicativo, suporta arquivos como: .mp3, .wav, .ogg.

Exemplo:
              
                <audio uma cacetada de atributos> <source src=""/> </audio> -> explicação do <source/> e dos atributos relacionados aos 2, mais a frente da página.

- video

Mesma coisa do de cima, só que né, para vídeos.

- picture

- source

Como demonstrado anteriormente, será utilizado juntamente de algum elemento para a anexação de um conteúdo, porém, terá como principal papel, realmente ditar qual será a extensão/conteúdo demontrado - um exemplo de sua funcionalidade, é, colocar diversos elementos desse em questão, para prevenir erros do navegador com a extensão do arquivo original.

Exemplo (utilizando audio):
              
                <audio controls>
                  <source src="LINK.ogg" type="audio/ogg" />
                  <source src="LINK.wav" type="audio/wav" />         -> todos os elementos e atributos mostrados, serão explicados posteriormente.
                  <source src="LINK.mp3" type="audio/mpeg" />
                </audio>

- svg

Utilizado para adicionar imagens vetoriais, utilizando formas matemáticas, a fim de evitar a perda de qualidade que se teria em imagens .jpg e .png - elemento usado principalmente para ícones, logosl, gráficos, pode ser manipulado por CSS e JavaScript, porém, é mais comum utilizar-se bibliotecas de ícones, como: Font Awesome.

- circle
- path

Ambos seriam filhos do <svg> que ajudariam na confecção da imagem vetorial.

- iframe

Anexa um site externo no seu próprio em uma janela reduzida, através de um link por exemplo - usos mais comuns incluem: vídeos do YouTube, e mapas do Google Maps

Exemplo:
              
                <iframe src="" title="" width="" height=""></iframe> -> os diversos atributos atribuídos ao <iframe> serão abordados futuramente em: Atributos.

- figcaption

Utilizado para adicionar uma descrição em uma mídia anterior a ela.

Exemplo:
              
                <figure>
                  <img/>
                  <figcaption>descrição</figcaption>
                </figure>

- select, option

Dois elementos utilizados para a criação de um menu selecionável.

Exemplo:
              
                <label for="carros">Escolha um carro:</label>
                <select id="carros" name="carros">
                  <option value="">X</option>
                  <option value="">Y</option>          -> Informações acerca dos atributos, será visto posteriormente em Atributos
                  <option value="">Z</option>
                </select>

- textarea

Utilizado para criar uma caixa de escrita de textos mais extensos, comentários por exemplo.

Exemplo:
              
                <label for="comentario">Comentário:</label>
                <textarea id="comentario" name="comentario" rows="5" cols="33" placeholder="Digite Aqui...">TEXTO</textarea>
                (Informações acerca dos atributos, será visto posteriormente em Atributos)

Alteradores de Texto

- em

Utilizado para dar ênfase em uma palavra ou frase, utilizando a característica de "itálico".

Exemplo:
              
                <em>frase destacada</em>

- strong

Utilizado para dar ênfase em uma palavra ou frase - geralmente de urgência ou importância - utilizando a característica de "negrito".

Exemplo:
              
                <strong>frase destacada</strong>

- font

Utilizado para determinar uma cor e o tamanho de uma palavra ou um texto.

Exemplo:
              
                <font size="" color="">Texto Modificado</font>

- center

Utilizado para centralizar textos horizontalmente.

Exemplo:
              
                <center>O Texto pode ser aqui<p>OU aqui</p></center>

- big

Como o nome indica, faz com que, o trecho ou termos dentro desse elemento, seja maior que o normal.

Exemplo:
              
                <p>Texto Texto<big>Texto Grande</big></p>

Obs: Apesar de existirem esses três últimos termos citados, que alterem significativamente o texto, eles são considerados termos obsoletos, visto que, linguagens de estilização, como o CSS, permitem uma maior organização e maior facilidade em alterações como essas.

Semânticos Parte 1 (Principais)

- div

Apesar de não ser um semântico de fato, por ter um papel semelhante (no sentido de servir como "contâiner" para os elementos), está posto aqui - e como já dito, serve para abrigar conjuntos de elementos, possibilitando através de atributos - class="" por exemplo - uma modificação por CSS ou até JS de uma forma mais global, entretanto, como dito, por não ser um elemento semântico de fato (ver mais sobre a definição de Elementos Semânticos em Explicações/Conceitos Importantes, será preferível na maior parte dos casos, o uso de elementos semânticos com usos específicos.

- main

Utilizado para representar o conteúdo principal do corpo de um documento HTML, elemento único - ou seja - deve ter apenas 1 por documento.

Exemplo:
                 
                <main>   
                  <h2>Eeveelutions</h2>           
                    <h3>Eevee Antes de Evoluir - Características Importantes e Observações</h3> 
                      <p>Texto</p>
                </main>

- section

Utilizado para separar seções em documento - dividir o documento em partes temáticas.

Exemplo:
              
                <section>Flareon</section>

                <section>Jolteon</section>

                <section>Vaporeon</section>

- figure

Utilizado para agrupar mídia e descrição - exemplo: imagens + descrição.

Exemplo:
              
                <figure>
                  <img/>
                  <figcaption>descrição</figcaption>
                </figure>

- footer

Utilizado para definir um rodapé em um site (a parte inferior do site com informações importantes).

Exemplo:
              
                <footer>
                  <p>Autor: XXX</p>
                  <p><a href="email">email"</a></p>
                </footer>

- nav

Utilizado para agrupar elementos relacionados a links de navegação.

Exemplo:
              
                <nav>
                <a href="">XXXX</a>
                <a href="">YYYY</a>
                <a href="">ZZZZ</a>
                </nav>

- header

Utilizado para definir o "cabeçalho" do documento.

- address

Utilizado para agrupar elementos relacionados a endereços, contatos, etc.

Semânticos Parte 2 (Importantes, mas não tanto quanto os da Parte 1)

- i

Usado geralmente em trechos que se tem uma língua diferente do resto do código por exemplo, logo o termo precisará de um destaque/explicação sobre a língua externa - visualmente, a palavra cercada por esse elemento, naturalmente terá sua escrita destacada em itálico.

Exemplo:
              
                <p>There is a certain<i lang="fr">je ne sais quoi</i> in the air.</p>

- b

Utilizado para destacar um certo trecho ou termo em negrito, destacando uma importância a mais naquela informação semanticamente.

- dl

Utilizado para engloblar/separar uma lista de descrição - formado por termos como <dt> e <dd> por exemplo.

- blockquote

Utilizado para separar uma citação do resto do texto, podendo ter atributos como cite="" por exemplo. Ele deixa o texto presente dele mais para o centro da tela horizontalmente.

- cite

Apesar de ser utilizado como um atributo em certas ocasiões, ele também é um elemento próprio, que tem como função destacar de onde foi tirada a citação anterior, deixa o termo em itálico.

Exemplo:
              
                <blockquote cite="x.com">CITAÇÃO</blockquote>
                <p>—AUTOR, <cite>LIVRO, ETC</cite></p>

- q

Similar ao <blockquote>, porém, é utilizado para trechos/frases dentro de paragráfos.

Exemplo:
              
                <p>ABCDE<q cite="url">FGHIJ</q></p>

- abbr

Utilizado envolta de abreviações - acrônimos, siglas, etc -, juntamente dele, geralmente se utiliza o atributo title="", para descrever o que aquela abreviação quer dizer.

Exemplo:
              
                <p><abbr title="HyperText Markup Language">HTML</abbr>TEXTO</p>

- br

Elemento vazio utilizado para quebra de linha, ou seja, pra separar uma informação da outra, em cenários relacionados a endereços principalmente.

Exemplo:
              
                <p> RUA XXXXXX <br />
                BAIRRO XXXXXX <br />
                CIDADE XXXXX </p>

- time

Utilizado para determinar um horário/data em específico, sendo muitas vezes acompanhado pelo atributo datetime="".

- sup

Utilizado principalmente para escrever fórmulas químicas ou equações matemáticas, onde, ele eleva um caractere visualmente.

- sub

Utilizado principalmente para fórmulas químicas, desnivelando para baixo um caractere específico.

- code

Utilizado para englobar partes do texto com valores de códigos de alguma linguagem de programação ou até próprios códigos, geralmente linhas curtas, tem como mudança visual, uma fonte de mono espaçamento.

- pre

Utilizado englobando o <code> caso seja mais de uma linha de código extenso presente dentro do dito cujo.

- u

Utilizado para sublinhar falhas ortográficas ou outras anotações não-textuais, como nomes próprios, palavras estrangeiras, etc.

- s

Utilizado para englobar frases ou trechos que não são mais tão relevantes - como se o documento tivesse passado por alguma alteração -, por padrão, ela "risca" o trecho englobado.

- ruby

Apresenta um pequeno trecho em cima de um texto principal, utilizado principalmente, para mostrar a pronúncia de termos do Leste Asiático, geralmente, junto a esse elemento, teremos como acompanhamento os próximos 2 elementos.

- rp

Utilizado como alternativa para navegadores que não suportam <ruby>.

- rt

Utilizado para indicar o texto das anotações do elemento <ruby>.

- article

Utilizado para retratar um conteúdo independente de uma página web, ou seja, onde esse trecho tem sentido próprio independente do restante do conteúdo da página.

- form

Pesquisar sobre.

- label

Utilizado junto do <input> para atribuir um rótulo/nome no site para tal elemento, onde a ligação entre os 2 elementos podem acontecer de 2 formas: de forma implícita - quando se usa os atributos for="" e id="" - ou de forma explicíta - quando o elemento de <input> é colocado dentro do elemento <label>.

- fieldset

Utilizado para agrupar elementos de <input>.

- legend

Utilizado junto do <fieldset>, para exemplificar sobre o que será o formulário abaixo.

Principais Atributos não-Booleanos

- src

Carrega um recurso externo (links geralmente) para dentro da página.

Exemplos de Possíveis Valores:
  • URL absoluta -> src="https://www.exemplo.com"
  • URL relativa -> src="imagens/foto.png"
Exemplo:
              
                <img src="img.com"/>

- alt

Usado para frases curtas, geralmente para descrever imagens por acessibilidade.

Exemplo:
                
                  <img src="" alt=descriçãozinha concisa da imagem/>

- href

Acopla um link em algo da página, fazendo com que, ao clicar, o usuário seja redirecionado a tal local.

Exemplos de Possíveis Valores:
  • href="tel:NUMERO"
  • href="mailto:EMAIL"
  • href="#algum id do código"
  • href="link externo"
Exemplo:
                
                  <a href="https://www.exemplo.com">palavra clicável</a>

- target

Define como um link vai ser aberto.

Exemplos de Possíveis Valores:
  • target="_blank" -> define que o link será aberto em outra página.
  • target="_self" -> define que o link será aberto na página ou no iframe em questão (é o valor padrão do atributo target)
  • target="_parent" -> quando em algo diferente de um elemento de âncora, define que o link será aberto no pai do iframe por exemplo.
  • target="_top" -> ignora todos os iframes que seria barreiras pro "_parent" e obrigatoriamente abre na página inteira.
Exemplo:
                
                  <a href="https://www.exemplo.com" target="_blank">XXXX</a>

- type

Usado geralmente junto do elemento de <input> para definir qual será o tipo de input utilizado.

Exemplos de Possíveis Valores:
  • type="text" (valor padrão)
  • type="password"
  • type="email"
  • type="search"
  • type="tel"
  • type="url"
  • type="number"
  • type="range"
  • type="checkbox"
  • type="radio"
  • type="date"
  • type="datetime-local"
  • type="month"
  • type="week"
  • type="time"
  • type="file"
  • type="color"
  • type="hidden"
  • type="image"
  • type="button"
  • type="submit"
  • type="reset"
Exemplo:
                
                  <input type="checkbox"/>

- rel

Utilizado para especificar o link acoplado com o código HTML.

Exemplo de Possíveis Valores:
  • rel="stylesheet" -> relaciona ao estilo CSS geralmente
  • rel="preconnect"
  • rel="icon"
Exemplo:
                
                  <link rel="stylesheet" href=".styless.css"/>

- id

Adiciona um identificar ÚNICO a um elemento HTML.

Exemplo:
                
                  <h1 id="identificador"> nesse caso por exemplo, onde você escrever um #identificador, será buscado o elemento <h1>

- class

Funciona semelhantemente ao id, porém, ele pode ser aplicado a mais de 1 elemento, facilitando customização por exemplo.

Exemplo:
                
                  <h1 class="class">
                  <p class="class">

- srcset

Basicamente, possibilita a colocação de diversos links relacionados a uma mesma mídia, porém, com tamanhos diferentes por exemplo - permitindo com que, o navegador ou o dispositivo do usuário, escolha a resolução mais compatível com o dispositivo.

Exemplo:
                
              <img 
                src="imagem-800.jpg" (800x600)
                srcset="
                imagem-400.jpg 400w, (400x300)
                imagem-800.jpg 800w, (800x600)
                imagem-1600.jpg 1600w (1600x1200)
                "
                sizes="(max-width: 600px) 400px, 800px" 
                width="800" height="600" alt="paisagem bonita" loading="lazy"
              />

- for

Utilizado para relacionar algum elemento a outro elemento através do seu id.

Exemplo:
                
                  <label for="ABC">XXXX</label> <input type="" id="ABC"/> -> isso, além de melhorar a acessibilidade, permite que o usuário, ao clicar no XXXX, ele seja automaticamente instruído a interagir com o <input/>

- placeholder

Utilizado geralmente com elementos associados ao <form> para fornecer um exemplo no campo que será escrito por exemplo.

Exemplo:
                
                  <input type="text" id="name" name="name" placeholder="Enter Your Name:"/>

Outros Atributos não-Booleanos

- property

Utilizado como atributo de um metadado principalmente, tem como função, facilitar o compartilhamento/visualização da sua página em outros sites.

Exemplos de Possíveis Valores:
  • property="og:title"
  • property="og:description"
  • property="og:image"
  • property="og:url"
  • property="og:type"
  • property="og:site_name"

Note que, juntamente desse atributo, se terá outro - o content="" - que indicará o conteúdo descrito pelo property="og:".

Exemplo:
                
                  <meta property="og:type" content="website"/>

- fill

Utilizado geralmente juntamente do elemento <svg> para representar qual será a cor preenchida.

- height e width

Define o tamanho horizontal e vertical de algum element - geralmente <img/> e <iframe>.

Exemplo:
                
                  <iframe src="" height="x" width="y"></iframe>

- allow

Define permissões de algum elemento, geralmente <iframe>.

Exemplos de Possíveis Valores:
  • allow="fullscreen"
  • allow="camera"
  • allow="autoplay"
  • allow="microphone"
  • allow="geolocation"
  • allow="clipboard-write"
Exemplo:
                
                  <iframe src="" height="" width="" allow="fullscreen"></iframe>

- loading

Define como será a forma na qual alguma mídia carrega, sendo mais comum entre imagens.

Exemplos de Possíveis Valores:
  • Único realmente importante por agora: loading="lazy" -> define que a imagem seja carregada apenas quando estiver próxima de ser visualizada.
Exemplo:
                
                  <img src="" loading="lazy"/>

- sizes

Define quanto que um conteúdo pegará da tela, tendo 3 formas de escrita principais: em pixels absolutos -> sizes="Xpx"; porcentagem -> sizes="Xvw"; condicional -> sizes="(max-width: Xpx) Ypx, Zpx": aqui diz que, caso a tela tenha até Xpx, o conteúdo ocupará Ypx, caso não, será Zpx

- referrerpolicy

Regula o quanto de informação um site-alvo anexado no seu site consegue do seu site.

Exemplos de Possíveis Valores:
  • Único importante por agora: referrerpolicy="strict-origin-when-cross-origin" -> faz com que seja mostrado apenas o domínio inicial para o site-alvo.

- cite

Valor sempre é uma URL, correspondente a da onde foi tirada uma citação.

- datetime

Utilizado muitas vezes junto do elemento <time>, para "formalizar" de certa forma, o tempo descrito, auxiliando o navegador por exemplo na busca do seu website. Escrita apoiada no ISO 8601.

Exemplo:
                
                  <time datetime="2024-06-15T15:00">June 15</time>

- action

Especifica para onde será mandado os dados obtidos por elemento associados ao <form> após o envio do usuário.

Exemplo:
                
                  <form action="submit.html">

- minlenght e maxlenght

Utilizado geralmente do <input> para representar a quantidade mínima e máxima de caracteres para o envio da informação coletada.

- method

Utilizado geralmente do <form> para especificar qual o HTTP a ser utilizar para o envio dos dados dos formulários. Sendo os valores mais comuns e importantes: GET e POST.

- name

Identificar os dados após o envio dos mesmos.

- size

- O que é um Elemento em HTML?

Tendo como significado HyperText Markup Language, é basicamente tudo o que tem de visível (e em certos casos, até fora tela) em uma página, ou seja, define a estrutura e principalmente o conteúdo de uma página - lembrando que, por ser uma linguagem de marcação, serve exclusivamente para isso: conteúdo e estrutura - coisas além disso, como: estilização e interatividade com a página, são funções de outras linguagens/códigos.

- Além disso, lembrar sempre da síntaxe padrão de um Elemento: <element>Conteúdo</element>

- O que é um Elemento Semântico?

É uma categoria de Elementos, que tem como principal função, uma melhor estruturação e organização da página web, além de fornecer diversos tipos de acessibilidade para o usuário - lembrando, porém, que a maioria dos Elementos presentes nessa categoria, não necessariamente mudam algo visualmente na página em questão, visto que o foco é a acessibilidade.

- O que é um Elemento Vazio?

É um tipo de Elemento que tem como principal característica, a falta de conteúdo próprio, e portanto, não se tem uma tag de fechamento comum - </element> - embora de poder ser escrito apenas como <element>, há quem utilize um / antes do > - não altera em nada na estrutura ou no funcionamento do código.

- O que é Aninhamento?

Basicamente é o que organiza o código onde elementos são subcategorias de elementos anteriores, através de espaços, por exemplo: 

            
              <element>
                <element2></element>
              </element>

- O que é relação de Pai e Filho entre Elementos?

Tendo como ponto de partida o tópico anterior - aninhamento - relação de pai e filho, é basicamente, quando se tem um elemento aninhado a outro, e, portanto, o elemento mais superior, seria o pai do elemento inferior aninhado a ele.

Exemplo:
            
              <element>                        -> seria o pai
                <element2></element>     -> filho
              </element>

- O que é uma entidade HTML?

São amontoados de caracteres usados para digitarmos de forma visual, algo que seria lido pela máquina como um código < -> isso por exemplo. Para representarmos isto em uma página, teremos 3 principais formas de tal: codificação baseado em &X;, onde X é o código específico; forma numérica, &#X, onde X é algum número; e por fim, utilizando uma codificação baseado na tabela ASCII -> &#X - onde X é um código em hexadecimal correspondente a um caractere na tabela ASCII.

- O que é um Atributo em HTML?

É basicamente algo que fornecerá uma característica, informação, tipagem, etc, à um elemento HTML específico, tendo sua colocação, no elemento de abertura - ou seja <elemento atributos> - além disso, a maioria dos atributos, sempre assumem um valor específico, correlacionado ao que você busca pro seu elemeneto HTML.

- O que é um Atributo Booleano?

São elementos que não necessitam de um valor para funcionarem, a presença deles em um elemento, já basta pro seu valor - de certa forma, seria como se quando ele está no elemento, o atributo já está valendo, e se ele estiver ausente, ele estaria "desligado", fazendo com que, não seja necessário um valor específico.

- Qual a diferença de um Caminho Relativo e um Caminho Absoluto?

Primeiramente, o conceito de Caminho no geral no contexto de códigos - basicamente, onde algo, geralmente uma mídia, está localizado - a partir disso, teremos 2 formas de identificar/específicar onde está localizado tal algo, o caminho relativo: onde ele começa a partir de onde está sendo realizado a busca, ou seja, a partir de onde será seu "destino final", por exemplo: se eu tenho um código em index.html, e tenho uma imagem a ser buscada para esse código, em uma pasta diferente da mesma, através de um caminho relativo, partindo do index.html, eu vou até a pasta onde está inserida tal imagem, exemplo: src="..src/image.png" (isso será visto de uma forma mais clara no exemplo abaixo). Já um caminho absoluto, em sua maioria, é uma URL comlpeta de algum website, entretanto, podendo ser descrita de certa forma como um caminho relativo, porém, diferentemente do caminho relativo, ele sempre partirá na raiz da coleção de arquivos.

Para se fazer esses caminhos, principalmente entre arquivos em uma pasta, se utiliza certos códigos para tal:

  • / -> é basicamente o que "separa" um arquivo do outro (imagens/foto.jpg -> significa entrar em imagens, e depois em foto.jpg)
  • ./ -> arquivo procurado está na pasta atual (apesar de sua ausência não ser algo tão necessário, visto que "imagem.jpg" e "./imagem.jpg" na maioria dos casos resulta no mesmo)
  • ../ -> voltar uma pasta

Exemplos Gerais:
        
          Tendo esse conjunto de pastas/arquivos: 
          my-app/
          ├─ public/
          │  ├─ favicon.ico
          │  ├─ index.html
          ├─ src/
          │  ├─ index.css
          │  ├─ index.js
                            -Iremos utilizar alguns casos para exemplificar a utilização de cada símbolo:
          1. Vamos supor que queremos colocar o favicon.ico em um atributo presente no index.html, como possível ver no esquema, os 2 fazem parte da mesma pasta, portanto, para podermos procurar o favicon.ico, apenas um "src="./favicon.ico"> é o suficiente.
          2. Agora vamos supor que queremos colocar dentro do index.html, o index.css, nesse caso, utilizaremos: src="../src/index.css">.
          3. E no caso de um URL absoluto (assumindo que queremos buscar o favicon.ico): src="/public/favicon.ico"

- O que é um Boilerplate?

É basicamente algo utilizado por quase todo programador, que é como se fosse um esqueleto base, onde já se tem coisas essenciais para quase todo e qualquer website, fazendo com que, o programador já parta de uma base sólida, sem a necessidade de quebrar cabeça com algo tão "comum/geral".

- Exemplo de Boilerplate

          <!DOCTYPE html>
          <html lang="en">
            <head>
              <meta charset="utf-8" />

              <meta name="description" content="">

              <meta property="og:title" content="">
              <meta property="og:description" content="">
              <meta property="og:type" content="website">
              <meta property="og:url" content="">
              <meta property="og:image" content="">
              <meta property="og:site_name" content="">

              <meta name="viewport" content="width=device-width, initial-scale=1.0" />
              <title></title>
              <link rel="stylesheet" href="./styles.css" />

            </head>

            <body>
            conteúdo da página
            </body>
          </html>
